feat(auth): implement Day 8 user authentication and profile system

## Backend Changes
- Add authentication routes to API Gateway (`/api/v1/auth/*`)
  - POST /auth/register - User registration with JWT
  - POST /auth/login - User login with access & refresh tokens
  - POST /auth/refresh - Token refresh mechanism
  - GET /auth/me - Get current authenticated user
  - PUT /auth/profile - Update user profile
- Integrate existing auth-service with API Gateway
- Auth-service includes JWT-based authentication, role-based access control (admin, volunteer, citizen, developer, school)

## Frontend Changes
- **AuthContext**: Global authentication state management
  - Auto-load user from localStorage on mount
  - Login, register, logout functions
  - Token management (access & refresh tokens)
- **Auth Pages**:
  - `/auth/login` - Login page with email/password form
  - `/auth/register` - Registration page with validation
  - `/profile` - User profile page with account info
- **UI Components**:
  - Navbar component with auth state display
  - Login/logout buttons based on auth status
  - User dropdown menu (username, profile link)
- **Integration**:
  - ReviewForm auto-fills username for authenticated users
  - Protected routes via AuthContext
  - JWT token sent with authenticated requests
